Got Questions?

Everything you need to know about Pristine.

Absolutely. Pristine runs entirely within your browser using the Web Canvas API and WebAssembly. Your images are loaded into memory locally, processed, and the output is offered for download — all without any network request. You can verify this by watching your network tab in DevTools: zero image data is transmitted.
Pristine supports PNG, JPEG/JPG, and WEBP formats — the three primary formats Gemini outputs images in. PNG is recommended for best results as it carries the alpha channel data needed for reverse compositing. Output is always delivered in lossless PNG.
No. Pristine uses Reverse Alpha Compositing — a mathematical inversion of how the watermark was applied. For pixels where the watermark is fully transparent (i.e., most of the image), the original pixel is recovered exactly. In regions of high watermark opacity, the reconstruction is visually imperceptible. Output is lossless PNG — no JPEG compression artifacts are introduced.
Yes! Batch processing is a first-class feature. Drop up to 100 images at once and Pristine will queue them for processing using Web Workers for true parallelism. A live progress indicator shows status per file. When complete, download all clean images as a single ZIP archive.
There is no catch. Pristine is free, open-source, and has no rate limits. It doesn't cost us server resources because everything runs in your browser. An optional Enterprise plan exists for teams who want advanced features, SLA guarantees, and dedicated support — but the core tool is free forever with no signup required.
